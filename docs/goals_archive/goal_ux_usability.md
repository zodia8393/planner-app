# /goal 모바일 UX 사용성 개선 — 실기기 스크린샷 기반

## 대상
/workspace/app_planners/my/ 전체 템플릿 → 완료 후 JM·Work 동기화 + Fly.io 배포

## 배경
Playwright 8뷰포트(280~500px) × 7페이지 × 인터랙션 상태 = 68장 스크린샷 정밀 검증 결과, 레이아웃 깨짐은 없으나 "보기 힘들다/쓰기 힘들다" 문제가 다수 발견됨. 아래는 깨짐이 아닌 사용성·가시성 문제만 정리한 것.

---

## 1. 할일(todos) — 폼 과부하 + 필터 혼잡

### 1-1. 필터 버튼 과밀
**현상**: 전체/미완료/완료/중요/높음/마감임박/완료보기/간편보기 — 8개 버튼이 상단에 나열. 280px에서 3줄, 360px에서도 2줄. 어디를 눌러야 할지 모름.
**개선**: 
- 1차 필터(전체/미완료/완료) 3개만 상단 노출
- 나머지(중요/높음/마감임박)는 "정렬·필터" 드롭다운 또는 시트로 이동
- 완료보기/간편보기는 토글 스위치로 분리

### 1-2. 폼 항상 펼쳐짐
**현상**: 제목/날짜/우선순위/카테고리/반복/에너지/태그 — 7개 필드가 항상 전부 보임. 간단한 할일 하나 추가하는데 스크롤해야 목록이 보임.
**개선**: 
- 기본: 제목 입력 + 추가 버튼만 노출 (1줄)
- "옵션 더보기" 탭하면 나머지 필드 아코디언 펼침
- 또는 FAB → 바텀시트 모달로 전환

### 1-3. "보통" 중복 혼동
**현상**: 우선순위 "보통"과 에너지 레벨 "보통"이 바로 옆에 있어 어느 "보통"인지 구분 불가.
**개선**: 에너지 레벨 필드에 라벨 "⚡에너지" 명시, 또는 에너지를 옵션 확장 영역으로 이동

---

## 2. 날짜 표시 혼란

### 2-1. 날짜 형식 불일치
**현상**: 동일 앱 내에서 3가지 형식 혼재
- `05/22/2026` (mm/dd/yyyy) — 메모, 업무일지 date picker
- `2026-05-22` — 할일 내부
- `05월 22일` — 대시보드
**개선**: 전체 `YYYY.MM.DD` 또는 `MM월 DD일` 한국어 형식으로 통일. date input의 placeholder도 `yyyy-mm-dd`로 변경

---

## 3. 다크모드 대비 부족

### 3-1. 보조 텍스트 가독성
**현상**: `text-slate-400` (#94a3b8) on `bg-slate-800/900` — 대비비 약 3.5:1로 WCAG AA(4.5:1) 미달. 메모 날짜, 업무일지 설명, 설정 부제목 등 보조 텍스트가 거의 안 보임.
**개선**: 보조 텍스트를 `text-slate-300` (#cbd5e1)으로 상향 → 대비비 약 5.5:1

### 3-2. 입력 필드 구분
**현상**: 다크 배경 위의 다크 input — 빈 입력 필드와 배경의 경계가 불명확. 특히 카테고리 드롭다운이 배경에 묻힘.
**개선**: input/select에 `border-slate-500` 또는 미세한 `ring-1 ring-slate-600` 테두리 추가

---

## 4. 오렌지 단색 피로

### 4-1. 모든 액션이 동일 오렌지
**현상**: 추가 버튼, FAB, 필터 active, 날짜 배지, 하단 네비 active, 토스트 — 전부 amber-600. 뭐가 중요하고 뭐가 보조인지 구분 불가.
**개선**:
- Primary 액션(추가/저장): amber-600 유지
- 보조 필터/토글: `bg-slate-600` 또는 `outline` 스타일 (선택 시에만 amber)
- 위험 액션(삭제): red 계열 유지
- 정보 배지(완료/대기): 별도 색상 (green-600/blue-500)

---

## 5. 업무일지(worklogs) — 폼 공간 낭비

### 5-1. 날짜 선택 과도한 공간
**현상**: 시작일/종료일 2개 date input이 각각 전체 너비 1줄씩 차지 → 280px에서 화면의 40%가 날짜 선택에 소모.
**개선**: 시작~종료를 한 줄에 배치 (flex row gap-2), 또는 "이번 주/지난 주" 퀵 선택 버튼만 노출하고 커스텀 기간은 접기

---

## 6. 하단 네비게이션

### 6-1. 아이콘+라벨 가독성
**현상**: 하단 nav 라벨(대시보드/할 일/캘린더/업무일지/더보기)이 text-xs로 매우 작음. 280px에서는 글자가 거의 안 보임.
**개선**: 라벨 폰트 `text-[10px]` → `text-xs`(12px) 최소 보장, 활성 탭은 font-bold로 강조

---

## 7. 메모(memos)

### 7-1. 빠른 액션 이모지 버튼
**현상**: 📌🔥⭐💡 등 카테고리 퀵버튼이 너무 작고 간격이 좁아 오터치 빈번.
**개선**: 이모지 버튼 min-size 36px + gap-2 간격 보장

---

## 8. 빈 상태 메시지 위치

### 8-1. 빈 상태 SVG가 폼 아래에 매몰
**현상**: 할일/메모 빈 상태 일러스트가 폼 영역 아래에 위치 → 폼 필드가 화면을 다 채워서 스크롤하지 않으면 빈 상태 안내가 안 보임. 신규 사용자가 "데이터가 없다"는 걸 인지 못함.
**개선**: 빈 상태일 때 폼을 최소화(제목만)하고 빈 상태 일러스트를 더 위로 올리기

---

## 작업 방식
- CSS 클래스 변경 + 약간의 HTML 구조 변경 (아코디언, 드롭다운 이동 등)
- JS 로직 최소 변경 (토글 open/close 정도)
- My 완료 → JM·Work 동기화 → Fly.io 배포

## 하지 말 것
- 기능 추가/삭제 (순수 UX 개선만)
- 색상 테마 전면 교체 (미세 조정만)
- 데스크탑 UI 변경
- 라우터/백엔드 로직 변경

## 검증
- Playwright 8뷰포트 재캡처로 before/after 비교
- WCAG AA 대비비 4.5:1 이상 (보조 텍스트)
- 할일 폼 기본 상태에서 목록이 스크롤 없이 보이는지
- 필터 버튼이 1줄에 수용되는지 (360px 이상)
- 10개 페이지 200 OK
