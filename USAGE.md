| 앱 | 용도 | 접속 | 인증 |

| **JM** | 공유 플래너 | https://jm-planner.fly.dev | 프로필 선택 (PIN 가능) |

## 주요 기능 사용법

### 1. 할일 관리

**기본 사용**
- 할일 목록 상단의 입력 폼으로 새 할일 생성
- 제목, 마감일, 우선순위(1~5), 카테고리, 에너지 레벨 설정 가능
- 체크박스 클릭으로 완료/미완료 토글

**에너지 레벨 필터**
- 각 할일에 에너지 레벨 지정: 🟢 저 / 🔵 중 / 🔥 고
- 할일 목록 상단의 에너지 필터 버튼으로 컨디션에 맞는 작업만 표시
- 기본값은 "중" — 고에너지 작업은 🔥, 저에너지 작업은 🟢 뱃지로 표시

**일괄 작업**
- "일괄" 버튼 클릭 → 할일 왼쪽에 체크박스 표시
- 여러 할일 선택 후 "일괄 완료" 또는 "일괄 삭제"

**서브태스크**
- 할일 상세에서 하위 작업 추가/관리

### 2. 집중 모드 (Pomodoro)

- 화면 우하단의 플로팅 타이머 버튼 클릭
- 프리셋: 25분 / 45분 / 60분 또는 커스텀 시간 입력
- 카테고리 지정 가능 (업무일지에 자동 기록)
- 타이머 실행 중 페이지 이동해도 localStorage에 보존
- 완료 시 업무일지에 자동으로 시간 기록

### 3. 시간 예산

- 설정 > 시간 예산에서 카테고리별 주간 목표 시간 설정
- 대시보드에서 이번 주 카테고리별 진행률 바 표시
- 업무일지 기록 시 자동으로 집계

### 4. 할일 템플릿

- 도구 > 할일 템플릿에서 자주 쓰는 할일 세트 저장
- "템플릿 적용" 클릭 시 저장된 할일이 자동 생성
- 기존 할일을 선택해서 템플릿으로 저장도 가능

### 5. 자동화 규칙

- 도구 > 자동화에서 규칙 생성
- 트리거: 매일 / 매주(요일 지정) / 매월(일자 지정)
- 액션: 지정한 제목/카테고리/우선순위로 할일 자동 생성
- 대시보드 방문 시 자동 실행 (당일 미실행 규칙만)

### 6. 양식 (폼 빌더)

**기본 양식 (프리셋)**
- 품의서: 기안/승인 요청 양식 (결재상태: 대기/승인/반려)
- 회의록: 회의 기록 양식 (참석자, 안건, 결정사항, 후속조치)
- 업무일지: 일일 업무 보고 양식
- "기본 양식" 뱃지가 표시되며, 수정/삭제 불가 (복제 가능)

**커스텀 양식**
- 양식 빌더에서 드래그앤드롭으로 필드 구성
- 필드 타입: 텍스트, 숫자, 날짜, 시간, 셀렉트, 텍스트영역
- 생성한 양식에 엔트리(제출물) 기록

### 7. 주간/월간 리뷰

- 분석 > 리뷰에서 기간(주/월) 선택
- 완료율, 카테고리별 시간 투자, 일별 완료 추이 차트
- 이전/다음 기간 탐색 가능

### 8. 감사 로그

- 관리 > 감사 로그에서 할일 변경 이력 확인
- 필터: 전체 / 할일 / 일정 / 메모
- 액션별 색상: 생성(초록) / 수정(파랑) / 완료(에메랄드) / 삭제(빨강)
- 수정 시 변경 전/후 값 상세 표시

### 9. 완료 히트맵

- 통계 또는 대시보드에서 GitHub 스타일 365일 히트맵 표시
- 날짜별 할일 완료 수를 색상 농도로 시각화

### 10. 마스터 대시보드 (Hub)

- localhost:8000 접속
- JM/My/Work 3개 플래너의 미완료 할일, 오늘 완료, 주간 근무시간 집계
- 향후 7일 통합 일정 표시
- 5분마다 자동 새로고침

## 배포

```bash
# JM/My — Fly.io
cd /workspace/app/planners
bash deploy.sh jm   # 또는 my

# Work — 로컬
cd /workspace/app/planners/work
python3 -m uvicorn main:app --host 0.0.0.0 --port 8001

# Hub — 로컬
cd /workspace/app/planners/hub
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000
```

## 기술 스택

- **백엔드**: FastAPI + Jinja2 + SQLite (WAL mode)
- **프론트엔드**: HTMX + Tailwind CSS 3.4.17 (빌드) + Chart.js
- **인프라**: Fly.io (JM/My), 로컬 서버 (Work/Hub)
- **PWA**: Service Worker + manifest.json (오프라인 캐시)
